# WGO-131 — `wgo statusline`: fast, local-only context for shell prompts

## Summary

Add a `wgo statusline` command purpose-built for shell prompts and the Claude
Code statusline: a single fast invocation that renders the current work context
as a compact line (or JSON) using only local state, never blocking on the
network. Add a `-C/--repo <dir>` global flag so the command can target a
directory without the caller `cd`-ing.

## Problem / Motivation

`wgo .` is the wrong tool for a prompt even after WGO-130:

- It performs a **synchronous GitHub call** on every invocation
  (`github.ListPRsForBranch`, `github.go:425`), so render latency depends on the
  network.
- It performs a **filesystem walk** of the parent directory for
  `showSiblings` (`dot.go:221`).
- It emits multi-line output and OSC8 hyperlinks intended for interactive
  reading.

A statusline re-renders constantly (on the Claude Code side, once per turn;
in a shell, once per prompt). It needs: one process spawn, sub-100ms, no
network in the hot path, and a single line. Because `wgo` exposes no such mode
and no way to target a directory, the current Claude Code integration has to
`cd "$cwd"` in a subshell, background-cache `gh pr view` itself, and `grep`
`~/.wgo/plan.md` — logic that belongs in `wgo`.

## Proposed Solution

### 1. `wgo statusline` command

Reuses `buildContext` from WGO-130 but in a **local-only** mode:

- PR/CI data comes from the on-disk cache (WGO-132) only; a cache miss renders
  no PR segment and (optionally) triggers a background refresh — it never
  blocks.
- Siblings are skipped unless `--siblings` is passed.
- Output is a single line.

Flags:

- `--json` — emit the `models.Context` (same schema as WGO-130) instead of a
  formatted line, for users who format in their own prompt code.
- `--format <template>` — Go text/template over the context, e.g.
  `'{{.Branch}}{{if .Dirty}}*{{end}} {{with .Ticket}}[{{.}}]{{end}} {{range .PRs}}#{{.Number}} {{.ReviewDecision}}{{end}}'`.
  Ship a sensible default template.
- `--no-color` — disable ANSI (auto-disabled when stdout is not a TTY).
- `--refresh` — force a synchronous PR/CI refresh (for a background warmer,
  not the hot path).

### 2. Global `-C/--repo <dir>` flag

Add a persistent flag on the root command that sets the working directory used
by context resolution, so `wgo -C /path statusline` works without `cd`. All
commands that call `os.Getwd()` should consult this flag first.

### 3. Reference prompt script

Ship `contrib/statusline.sh` (and a fish/zsh snippet) that calls
`wgo -C "$dir" statusline` so downstream users do not hand-roll fragile shell.

## Acceptance Criteria

- [ ] `wgo statusline` renders a single line in <100ms with a warm cache and
      makes **zero** network calls in the default (non-`--refresh`) path.
- [ ] `wgo statusline --json` emits the WGO-130 schema.
- [ ] `wgo statusline --format '<tmpl>'` renders a Go template over the context.
- [ ] `wgo -C <dir> statusline` resolves context for `<dir>` without the caller
      changing directories; the flag works for `wgo .` too.
- [ ] With no cached PR data, the command still succeeds and simply omits the
      PR segment (graceful degradation), optionally kicking off a background
      refresh.
- [ ] `contrib/statusline.sh` is present and documented in the README.

## Out of Scope

- The cache implementation itself (WGO-132).
- Which PR fields are available (WGO-133).
