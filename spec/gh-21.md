# gh-21: Replace git with jj-vcs

**Status:** In progress (branch `jj2-vcs-update`)
**Issue:** https://github.com/dmihalcik-virtru/wgo/issues/21

## Goal

Replace wgo's git CLI backend with [Jujutsu (jj-vcs)](https://jj-vcs.github.io/jj/) as the sole VCS. Preserve wgo's structured GitHub PR stacking and spec-driven development features while shedding hundreds of lines of fragile git rebase orchestration in favor of jj's native auto-restacking DAG.

## Why jj

- **First-class workspaces** map cleanly onto wgo's worktree model with no `index.lock` contention ā€” enabling parallel AI agents to operate in isolated folders against the same repo without VCS lock contention.
- **Auto-restacks descendants** when an ancestor commit changes, eliminating wgo's complex rebase orchestration (checkpoints, force-with-lease, conflict reports, `.git/rebase-merge` state detection).
- **Anonymous, stable change IDs** mean wgo's shadow stack identity layer (`Stacks[]`, `Annotation.Parents`, `StackID`) becomes unnecessary ā€” the jj DAG is the single source of truth.
- **Conflict commits** instead of interactive merge blocks: if an agent hits a merge conflict, jj records a "conflict commit" and lets the agent keep running.
- **`jj undo`** rolls back the entire repository state instantly.

## Constraints (user-confirmed)

1. **Full replacement**, single PR. `internal/git` deleted entirely; `internal/jj` is the only backend.
2. **No colocate**. Pure jj repos only ā€” `jj git init` without `--colocate`. Drop all functionality that requires colocate.
3. **Delete shadow state immediately**. Remove `State.Stacks`, `Annotation.Parents`, `Annotation.StackID`. Query the jj DAG as single source of truth. Bump `State.Version` to 2 with no migration path ā€” old state files refused with a clear error pointing users at deletion.
4. **No migration tooling**. Existing git-only users start over (documented in README).
5. **No git CLI dependency**. Replace `internal/github` shell-outs to the `gh` CLI with direct HTTPS calls to api.github.com, since `gh` requires a `.git/` directory which pure jj repos lack.
6. **Drop the hook system entirely**. No more `~/.wgo/hooks/` or `core.hooksPath` chaining; no `wgo hooks` or `wgo _event` commands. Spec enforcement moves to an active `wgo doctor` check.
7. **Drop cross-repo stacking**. `wgo sync` operates per-repo. Multi-repo `wgo add` still creates parallel workspaces, but stacks don't span repos.
8. **Real jj in all tests**, no mocking. CI installs jj.
9. **No jj version pinning** ā€” trust the user. Smoke test detects template-schema drift loudly.

## User-facing commands

| Command | Behavior |
|---|---|
| `wgo add TICKET` | Runs `jj workspace add` to create `${dev_root}/TICKET/{org}/{repo}` based on the current workspace's tracked bookmark. Creates anonymous commit, generates spec scaffold. Multi-repo loop unchanged. |
| `wgo to <PR URL>` | `jj git fetch --branch glob:pull/N/head`; creates workspace at `${dev_root}/pr-N-slug/{org}/{repo}`; checks out the PR's bookmark. |
| `wgo to owner/repo[@branch]` | `jj git clone` if missing; `jj workspace add` on default-bookmark revset. |
| `wgo sync` | Replaces `wgo stack restack` and `wgo stack sync`. Per-repo: queries jj DAG for ancestor bookmarks with open PRs, calls GitHub API to retarget child PR bases, regenerates markdown stack markers. jj auto-restacks descendants when ancestors edit ā€” no manual restack needed. |
| `wgo doctor` | NEW. Walks tracked repos, runs spec-presence/staleness checks (replacing passive pre-commit enforcement). Exits non-zero with `--strict`. |
| `wgo ls`, `wgo clean`, `wgo plan`, `wgo today`, `wgo status`, `wgo spec`, `wgo dot`, `wgo done`, `wgo pr`, `wgo join`, `wgo track`, `wgo contrib`, `wgo pilot` | Rewired: git queries replaced with jj queries, gh-CLI shell-outs replaced with HTTP calls. Behavior preserved. |

## Removed commands

- `wgo hooks {install, uninstall, status}` ā€” hook system deleted entirely
- `wgo _event ...` ā€” no more passive event recording
- `wgo stack restack` ā€” jj auto-restacks
- `wgo stack {new, push, rm, adopt}` ā€” jj's DAG is authoritative; no separate stack identity records
- `wgo stack sync` ā†’ renamed to `wgo sync`
- `wgo stack status` ā†’ may stay as read-only view, or rename to `wgo dag`

## Architecture

### `internal/jj`

The new VCS backend. Client interface (~30 methods) wraps `jj` CLI; CLIClient shells out and parses output via a single JSON-escape template (`jj log -T <template>`). Pinned template schema version; smoke test fails loudly on upstream drift.

Methods cover: workspaces (Add/Forget/List), bookmarks (Create/Set/Delete/List), DAG queries (Log/Resolve/CurrentChange/Status), mutations (New/Describe/EditChange), git interop (Init/Clone/Fetch/Push).

**Deleted git operations with no jj equivalent**: Rebase, RebaseOnto, Merge, RebaseContinue, HasActiveRebase, PushForceWithLease, ForceLeaseRef. jj auto-restacks; `jj git push` enforces lease semantics natively.

### `internal/github` (rewritten)

Direct REST API client. Uses `gh auth token` (with `GITHUB_TOKEN` fallback) for credentials only ā€” no shell-out to gh for repo operations. ETag-aware caching via `internal/cache`, 60s TTL, X-RateLimit honored. Public Client interface unchanged so callers don't rewire.

### `internal/store` (slimmed)

`Annotation`: `{Purpose, SpecPath, SpecState, CreatedAt, UpdatedAt}` ā€” `Parents`, `StackID` deleted.
`State`: keeps `Version` (bump to 2), `Repos`, `Annotations`, `Efforts`, `AgentSessions`. `Stacks` and `Stack` struct deleted.
Annotation key semantics unchanged: `repo:bookmark` (was `repo:branch`).

### `internal/sync` (replaces `internal/stack`)

`internal/stack/restack.go` deleted entirely (Checkpoint, GitOps, Restack, rebaseOne, force-with-lease plumbing ā€” 1000+ LOC gone).

New `internal/sync/`:
- `marker.go` ā€” moved from `internal/stack/marker.go`; PR body block serializer unchanged
- `graph.go` ā€” builds Graph from `jj log -r 'bookmarks() & ::heads()' -T <template>`, not from `Annotation.Parents`
- `sync.go` ā€” per-repo algorithm:
  1. `jj git fetch`
  2. Build graph from jj log
  3. For each bookmark with open PR, walk ancestors to find nearest ancestor with open PR ā†’ desired base
  4. If PR.BaseRefName ā‰  desired, call `gh.UpdatePRBase`
  5. Regenerate marker, `gh.UpdatePRBody` if changed

### `internal/discovery`

Switch `.git/` detection to `.jj/`. Drop file-vs-directory worktree distinction (jj's `.jj` is always a directory). Replace git-worktree-list logic with `jj.WorkspaceRoot` + `jj.ListWorkspaces`.

### Deleted

- `internal/git/` (entire package)
- `internal/stack/restack.go`, `restack_test.go`
- `internal/hooks/` (entire package)
- `internal/cmd/{hooks,event,event_test}.go`
- `scripts/hooks/`, `scripts/setup-local-hooks.sh`
- `internal/cmd/stack.go` subcommands: restack, new, push, rm, adopt

## Acceptance criteria

- `go build ./...` succeeds with no git CLI dependency.
- `go test ./...` passes against real jj (jj must be on PATH).
- `wgo add TICKET` creates a jj workspace + bookmark on the current workspace's tracked bookmark.
- `wgo to <PR URL>` fetches and checks out a PR in a new workspace under `${dev_root}/pr-N-slug/{org}/{repo}`.
- `wgo sync` walks the jj DAG, retargets child PR bases via GitHub HTTP API, regenerates markers ā€” without any local git operations.
- `wgo doctor` enforces spec presence on tracked workspaces (active check, exits non-zero with `--strict`).
- `internal/git/` directory does not exist.
- `internal/hooks/` directory does not exist.
- No `exec.Command("git", ...)` calls remain in the codebase.
- No `exec.Command("gh", ...)` calls remain except `gh auth token` in `internal/github` for credential bootstrap.
- README documents jj prerequisite, list of removed features, and the no-migration policy.

## Out of scope

- jj version pinning or runtime version check
- Migrating existing `~/.wgo/state.json` from version 1 to version 2 (users delete and start over)
- Pre-commit-style passive enforcement (replaced by active `wgo doctor`)
- Cross-repo stacked PRs (per-repo sync only)
- Colocated jj repos
- Restoring any deleted command (`wgo stack restack`, etc.)

## Risks

1. **jj bookmarks don't auto-advance.** After `jj new` on a bookmark tip, the bookmark stays put ā€” `jj git push` will refuse. Any wgo mutation command that creates commits must also call `jj.BookmarkSet(name, "@-")` before push. Users running `jj` directly need to do this themselves; document in README.
2. **`jj log` template stability is not guaranteed.** Pin a schema version constant; smoke test fails loudly on drift.
3. **Loss of passive spec enforcement.** Users must run `wgo doctor` (manually or in CI). Update `Spec-driven Team Development Pilot.md`.
4. **No git CLI at all** means external tools that need `.git/` (IDE git plugins, `pre-commit` framework, `direnv` git features) won't work in wgo workspaces. Document in README's tradeoffs section.
5. **Cross-repo stacking dropped.** Multi-repo `wgo add` still works, but `wgo sync` is per-repo.
6. **GitHub auth via `gh auth token`.** Zero-config if `gh` is set up; otherwise `GITHUB_TOKEN` env var. No interactive flow.
7. **`jj git push` lease errors are stderr text, not structured.** `internal/jj` push wrapper must parse and wrap with a typed error.
