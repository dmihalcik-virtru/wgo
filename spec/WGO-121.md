# WGO-121 — `wgo spec jira`: Jira integration for spec sync and context

## Summary

Add a `wgo spec jira` command family that uses `acli jira` to sync local spec files with their Jira tickets — pulling title, status, priority, comments, and stakeholders without requiring direct Jira API credentials.

## Problem / Motivation

Specs are created at ticket-start and then edited locally, but the source of truth for priority, status, and discussion is Jira. Today there is no path from Jira back into the spec:

- Ticket priority changes (e.g. deprioritized) are invisible in the spec.
- Comments and decisions logged in Jira never make it into the spec body.
- Watchers and assignees added in Jira are not reflected in `authors:`.
- A developer switching context must open Jira in a browser to understand the current state of a ticket they are about to work on.

The `acli` Atlassian CLI is already authenticated and installable; wrapping it avoids storing additional credentials in `wgo`.

## Proposed Solution

Four new subcommands under `wgo spec jira`, all resolving the ticket the same way as other spec commands (explicit arg → current branch → error):

### `wgo spec jira focus [TICKET]`
Read-only. Prints the spec body, then a divider, then the Jira issue summary, status, priority, assignee, and up to 10 recent comments. Gives the developer a single-pane view before starting work on a ticket.

### `wgo spec jira sync [TICKET]`
Writes spec frontmatter. Fetches `summary`, `status`, and `priority` from Jira and updates:
- `title` ← Jira summary
- `status` ← mapped from Jira status (see mapping below)
- `jira_priority` ← Jira priority name (new optional frontmatter field)
- `updated` ← bumped to today

Jira → spec status mapping (case-insensitive substring match):
| Jira status contains | Spec status |
|---|---|
| "To Do", "Open", "Backlog" | `draft` |
| "In Progress", "In Review" | `in_progress` |
| "Done", "Closed", "Resolved" | `shipped` |
| "Won't Do", "Won't Fix", "Abandoned" | `abandoned` |
| anything else | unchanged |

### `wgo spec jira pull [TICKET]`
Writes spec body. Fetches up to 20 recent comments from Jira. Appends or replaces a `## Jira Notes` section at the end of the spec with the formatted comments and a sync timestamp. Body text is extracted from Jira's ADF format by collecting `text` leaf nodes recursively.

### `wgo spec jira users [TICKET]`
Writes spec body. Fetches the ticket's assignee and watchers. Deduplicates by account ID. Appends or replaces a `## Stakeholders` section in the spec body with a formatted list (display name, email). Does not modify `authors:` frontmatter automatically; the developer can move entries manually when appropriate.

## Inputs / Outputs / Contracts

### New package: `internal/jira/client.go`

```go
package jira

// CheckAuth runs `acli jira auth status` and returns an error if not authenticated.
func CheckAuth() error

// GetIssue fetches issue fields via `acli jira workitem view KEY --json`.
func GetIssue(ticket string) (*Issue, error)

// GetComments fetches up to limit comments via `acli jira workitem comment list --key KEY --json`.
func GetComments(ticket string, limit int) ([]Comment, error)

// GetWatchers fetches watchers via `acli jira workitem watcher list --key KEY --json`.
func GetWatchers(ticket string) ([]User, error)

type Issue struct {
    Key    string
    Fields IssueFields
}
type IssueFields struct {
    Summary  string
    Status   struct{ Name string }
    Priority struct{ Name string }
    Assignee *User
}
type Comment struct {
    ID      string
    Author  User
    Body    string    // ADF extracted or plain text
    Created time.Time
}
type User struct {
    AccountID    string
    DisplayName  string
    EmailAddress string
}
```

All functions shell out to `acli`; stderr is captured and included in error messages to surface auth failures clearly.

### New file: `internal/cmd/spec_jira.go`

Defines `specJiraCmd` (parent) and four leaf commands registered in `init()` under `specCmd`.
Uses `resolveSpecTarget` (existing, in `spec.go`) for ticket resolution.
Uses `spec.FindByTicket`, `spec.Parse`, `spec.UpdateFrontmatter` for file I/O.

Section replacement pattern for `pull` and `users`:
```
body_before = strings.SplitN(body, "\n## <Section>", 2)[0]
new_body = body_before + "\n" + rendered_section
```
Writes via `.tmp` + `os.Rename` (matching existing atomic-write pattern).

### Spec frontmatter extension (`internal/spec/spec.go`)

```go
JiraPriority string `yaml:"jira_priority,omitempty"`
```

### Config extension (`internal/config/config.go`)

```go
type JiraConfig struct {
    Project string `mapstructure:"project"` // default project key, e.g. "WGO" (informational only)
}
// Added to Config:
Jira JiraConfig `mapstructure:"jira"`
```

A commented `[jira]` block is added to `createDefaultConfig`.

## Edge Cases & Constraints

- **`acli` not installed or not authenticated**: `CheckAuth()` returns a clear error; all four commands call it first and print a helpful message pointing to `acli jira auth login`.
- **Ticket has no Jira issue**: `GetIssue` returns a wrapped error; command prints `ticket WGO-XXX not found in Jira` and exits non-zero.
- **ADF body**: Jira returns rich text as ADF JSON. `extractADFText` walks the `content` tree collecting `"text"` leaf values. Falls back gracefully if body is a plain string.
- **`acli` comment JSON format**: `acli` may return `[...]` or `{"comments":[...]}`. Parse both forms by trying array unmarshal first.
- **Existing `## Jira Notes` / `## Stakeholders`**: replaced in-place, not duplicated.
- **No spec file**: commands error with `no spec for TICKET — run: wgo spec new TICKET` (same as other spec commands).
- **`jira_priority` in frontmatter**: `omitempty` means old spec files without this field round-trip with no diff.

## Out of Scope

- Writing back to Jira (no `acli jira workitem edit` calls).
- Automatically mapping Jira display names to GitHub handles.
- Polling / watch mode for Jira changes.
- Support for Jira Software board or sprint commands.
- Adding `jira_priority` or `jira_status` to the spec template (they are sync-only fields).

## Acceptance Criteria

- [ ] `wgo spec jira focus WGO-XXX` prints spec body, then Jira issue summary/status/priority/assignee, then up to 10 comments; exits 0 when authenticated, non-zero (with guidance) when not.
- [ ] `wgo spec jira sync WGO-XXX` updates `title`, `status`, and `jira_priority` in spec frontmatter and bumps `updated`; prints a one-line confirmation showing what changed.
- [ ] `wgo spec jira pull WGO-XXX` appends or replaces `## Jira Notes` in the spec body with up to 20 comments and a sync timestamp; second run replaces the previous section rather than duplicating it.
- [ ] `wgo spec jira users WGO-XXX` appends or replaces `## Stakeholders` in the spec body listing assignee and watchers; second run replaces rather than duplicates.
- [ ] All four commands accept an optional TICKET argument and fall back to parsing the current branch name when omitted.
- [ ] All four commands print a clear error and exit non-zero when `acli` is not authenticated.
- [ ] `go test ./internal/jira/... ./internal/spec/...` passes.
- [ ] `go build ./cmd/wgo` succeeds with no new warnings.
