# WGO-124 — wgo clean: authoritative merged-branch detection

## Summary

`wgo clean` blocks deletion of local branches for merged PRs whenever the branch tip is not reachable from `origin/<default>`. This causes false positives for squash merges, rebase merges, and branches whose remote counterparts were deleted after merge. 

This spec unifies multiple safety checks into an authoritative "proof-based" evaluation. A merged PR branch is considered safe to force-delete if it can be proven that the local tip contains no commits that weren't already pushed as part of the PR or merged into the default branch.

## Problem / Motivation

The current check treats "not reachable from `origin/main`" as equivalent to "local-only work would be lost." This is too coarse for modern GitHub workflows:
- **Squash Merges:** Create a new commit on main; original feature commits are never reachable from `origin/main`.
- **Rebase Merges:** Rewrite commit SHAs; reachability is lost.
- **Deleted Remotes:** After merge, GitHub often auto-deletes the remote branch, so `origin/<branch>` is gone.

The safety rule should ask: *Has this branch been merged AND does the local branch contain any commits NOT present in the merged result or the last pushed PR state?*

## Proposed Solution

Extend `PRInfo` with authoritative metadata from GitHub and use multiple "proof bases" in `wgo clean` to verify safety.

### 1. Authoritative Proof Bases
For a branch where `PRInfo.IsMerged() == true`, `wgo clean` will try to prove safety using these bases (in order):

1.  **Reachability from Target:** Is the local branch tip already reachable from `origin/<default>`? (Covers standard merges).
2.  **Merge Commit Ancestry:** Is the PR's `mergeCommit.oid` reachable from `origin/<default>`? (Covers squash/rebase merges where the landing commit is known).
3.  **Pushed PR Tip:** Does the local branch tip match (or is an ancestor of) the PR's `headRefOid`? (Proves no local commits were added after the last successful push to the PR).
4.  **Upstream Tip:** If the remote branch still exists locally (`origin/<branch>`), is the local branch reachable from it? (Covers general "no unpushed work" safety).

If any proof base confirms safety, `force = true` is set for the deletion. If none can prove safety, `wgo clean` remains conservative and returns a detailed error explaining exactly *why* deletion was blocked, citing missing upstream refs, unpushed commits, or missing local merge commits.

### 2. Merge Queues
GitHub Merge Queue handles the merge process. Once a PR passes the queue and is merged, its state becomes `MERGED`. Our proof bases (especially Merge Commit Ancestry and Pushed PR Tip) will correctly identify these as safe once the merge is finalized, even if the intermediate merge queue branch was deleted.

### `internal/cmd/clean.go`
Refactor `executeRemoval` for `KindLocalBranch`:

```go
case cleanup.KindLocalBranch:
    force := cleanForce
    if !force && c.PRInfo != nil && c.PRInfo.IsMerged() {
        defaultBranch, _ := gitClient.DefaultBranch(c.RepoPath)
        if defaultBranch == "" { defaultBranch = "main" }
        target := "origin/" + defaultBranch

        // Check 1: Reachable from target (Standard merge)
        hasExtra, _ := gitClient.HasLocalOnlyCommits(c.RepoPath, c.Branch, target)
        if !hasExtra { force = true }

        // Check 2: Merge Commit landed on target (Squash/Rebase)
        if !force && c.PRInfo.MergeCommit != nil && c.PRInfo.MergeCommit.OID != "" {
            sha := c.PRInfo.MergeCommit.OID
            isAnc, ancErr := gitClient.IsAncestor(c.RepoPath, sha, target)
            if ancErr != nil {
                // Try fetching if the commit isn't local
                if fetchErr := gitClient.Fetch(c.RepoPath); fetchErr == nil {
                    isAnc, _ = gitClient.IsAncestor(c.RepoPath, sha, target)
                }
            }
            if isAnc { force = true }
        }

        // Check 3: Local tip matches pushed PR head
        if !force && c.PRInfo.HeadSHA != "" {
            hasExtra, _ := gitClient.HasLocalOnlyCommits(c.RepoPath, c.Branch, c.PRInfo.HeadSHA)
            if !hasExtra { force = true }
        }

        // Check 4: Local tip matches upstream ref
        if !force {
            if upstream, _ := gitClient.UpstreamRef(c.RepoPath, c.Branch); upstream != "" {
                hasExtra, _ := gitClient.HasLocalOnlyCommits(c.RepoPath, c.Branch, upstream)
                if !hasExtra { force = true }
            }
        }

        if !force {
            // Provide a detailed error message indicating which checks failed.
            var reasons []string
            reasons = append(reasons, fmt.Sprintf("not reachable from %s", target))
            if c.PRInfo.MergeCommit != nil && c.PRInfo.MergeCommit.OID != "" {
                reasons = append(reasons, fmt.Sprintf("PR merge commit (%s) not found locally or not in %s", c.PRInfo.MergeCommit.OID[:7], target))
            }
            if c.PRInfo.HeadSHA != "" {
                reasons = append(reasons, fmt.Sprintf("local commits exist beyond pushed PR head (%s)", c.PRInfo.HeadSHA[:7]))
            }
            upstream, _ := gitClient.UpstreamRef(c.RepoPath, c.Branch)
            if upstream == "" {
                reasons = append(reasons, "no upstream branch configured")
            } else {
                reasons = append(reasons, fmt.Sprintf("local commits exist beyond upstream (%s)", upstream))
            }
            
            return fmt.Errorf("cannot verify branch is safe to delete:\n- %s\nuse --force to delete anyway", strings.Join(reasons, "\n- "))
        }
    }
    return gitClient.DeleteBranch(c.RepoPath, c.Branch, force)
```

## Edge Cases & Constraints

- **Fetch Requirement:** If `mergeCommit.oid` is not found locally, `wgo clean` will perform a single `git fetch --all --prune` and retry the ancestry check.
- **Unpushed Commits:** Genuinely unpushed local commits will fail all 4 checks, correctly blocking auto-deletion.
- **Multiple Remotes:** `wgo` generally assumes `origin` is the primary remote for PR tracking.
